# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is an academic research repository for a PhD dissertation project titled "Die Ästhetik der Plausibilität: Eine praxis-basierte Medienarchäologie algorithmischer Bildvervollständigung" (The Aesthetics of Plausibility: A Practice-Based Media Archaeology of Algorithmic Image Completion). The project investigates inpainting algorithms in digital panorama creation through systematic analysis of their artifacts.

## Repository Structure

- `Expose/` - Contains various versions of the dissertation exposé documents (v1-v6_current)
- `prd.md` - Product Requirements Document for an Academic Research Assistant MCP Server
- `How-to-Claude-Code-Tutorial.md` - Comprehensive tutorial for Claude Code usage
- Various research documents including framework evaluations and literature research

## Research Context

This is a media archaeology and critical algorithm studies project that:
- Analyzes algorithmic "errors" as epistemological objects
- Develops a practice-based methodology for algorithm criticism
- Plans to build an MCP server for academic research assistance
- Combines theoretical work with practical implementation

## Planned Development Work

Based on the PRD, the project plans to develop:

1. **Academic Research Assistant MCP Server** - A tool for PhD students to interact with their research library through Claude
2. **Forensic Analysis Tools** - Tools for detecting and analyzing inpainting artifacts
3. **Alternative Algorithm Implementations** - Counter-algorithms that challenge conventional completeness logics

## Development Environment

When this project transitions to active development, it will likely use:
- **Languages**: Python (for computer vision work), JavaScript/TypeScript (for MCP server)
- **Key Libraries**: OpenCV, scikit-image, NumPy for image processing
- **MCP Framework**: For Claude integration
- **Research Tools**: Git for version control, Jupyter notebooks for analysis

## Academic Standards

- Prioritize reproducibility and documentation
- All algorithms should be well-documented with academic citations
- Code should serve research goals, not just functionality
- Maintain clear separation between theoretical analysis and practical implementation
- Follow open science principles where possible

## Working with Research Documents

- The `Expose/` directory contains the evolution of the dissertation proposal
- `Expose-v6_current.md` is the most recent version
- When referencing the research, use the current exposé version
- The PRD represents a concrete implementation plan for part of the research

## Special Considerations

This is primarily a research repository focused on critical algorithm studies. When developing code:
- Consider the theoretical implications of implementation choices
- Document not just what code does, but why specific approaches were chosen
- Be prepared to implement "alternative" or "counter" algorithms that challenge conventional approaches
- Maintain academic rigor in both code and documentation
