# Rules

Es ist ein practice based phd expose. Ich bin Designer und kann code lesen und schreiben.

##Methodische Grundregeln

1. Praxis als Erkenntnismodus

Algorithmen nicht nur analysieren, sondern rekonstruieren, modifizieren, brechen
Code als primäre Quelle und kulturelles Artefakt behandeln
Eigene Implementierungen als Forschungsmethode

2. Artefakte als epistemische Objekte

Fehler in der Bildgenerierung als Erkenntnisfenster, nicht als Defizite
Minimale optische Ereignisse → größere techno-politische Narrative (Farocki/Paglen-Methode)
Systematische Artefakt-Produktion durch strategisches Bildmaterial

3. Fokus Algorithmen statt Plattformen

Open Source ermöglicht Forschungstiefe
Plattformen = instabile, personalisierte Objekte
Algorithmen = stabile, untersuchbare "cognitive infrastructures"

##Inhaltliche Grundregeln

4. Panorama als historisch-kulturelles Konzept

Nicht auf technische Sphere-Definition begrenzen
Globale Panorama-Genealogie erforschen (nicht nur westlich)
Alternative Totalitätskonzepte einbeziehen

5. Multiple Analyseebenen

Patent-Archäologie (Militär → Zivil Migration)
Standards & Metriken (historische Bewertungskriterien)
Code-Forensik (Comments, Magic Numbers, Git-History)
Interviews mit Entwickler\*innen (falls möglich)

6. Strategisches Test-Design

Kuratiertes Bildkorpus das Grenzen provoziert
Mathematische Edge Cases (Pole, 0% Overlap)
Kulturelle Grenzfälle
Historische Panoramen

## Narrative Grundregeln

7. Vom Detail zum System

Jeder Artefakt erzählt drei Geschichten: technisch, symptomatisch, kulturell
Mikro-Beobachtungen → Makro-Aussagen über visuelle Episteme
"Ästhetik der Plausibilität" als roter Faden

8. Kritische Technikgeschichte

Evolution als kontingente Entwicklung, nicht Fortschritt
Alternative Pfade sichtbar machen
Denaturalisierung scheinbar universeller Standards
