## PR Creation Process:
1. **Branch Management**
   - Create feature branch with descriptive name
   - Ensure all changes are committed
   - Push branch to remote repository
   - Verify branch is up-to-date with main

2. **Analyze Changes**
   - Review all modified files
   - Understand the scope of changes
   - Identify breaking changes or dependencies
   - Verify tests pass and build succeeds

3. **Generate PR Description**
   - Create clear, descriptive title
   - Write comprehensive summary of changes
   - List key features or bug fixes
   - Include testing instructions
   - Add screenshots/examples if applicable

4. **Set PR Metadata**
   - Assign appropriate reviewers
   - Add relevant labels (feature, bugfix, etc.)
   - Link related issues or tickets
   - Set milestone if applicable

5. **Quality Checks**
   - Ensure code follows project standards
   - Verify all tests pass
   - Check for merge conflicts
   - Validate CI/CD pipeline success

6. **Create PR**
   - Open pull request with generated content
   - Notify relevant team members
   - Provide PR URL for tracking
