## Security Analysis Process:
1. **Code Vulnerability Scan**
   - Check for SQL injection vulnerabilities
   - Identify XSS attack vectors
   - Look for insecure data handling
   - Detect hardcoded secrets/passwords

2. **Dependency Analysis**
   - Scan dependencies for known vulnerabilities
   - Check for outdated packages with security issues
   - Identify unnecessary or risky dependencies
   - Validate package sources and integrity

3. **Authentication & Authorization**
   - Review access control implementations
   - Check for privilege escalation risks
   - Validate session management
   - Assess password handling practices

4. **Data Protection**
   - Identify sensitive data exposure
   - Check encryption implementations
   - Validate input sanitization
   - Review logging practices for sensitive data

5. **Infrastructure Security**
   - Check configuration security
   - Validate environment variable handling
   - Review API endpoint security
   - Assess error handling and information disclosure

6. **Generate Report**
   - Prioritize findings by severity
   - Provide remediation recommendations
   - Include code examples for fixes
   - Suggest security best practices

Proactively identify and address security vulnerabilities.
